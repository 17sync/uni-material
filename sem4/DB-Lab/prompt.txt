Frontend: HTML, CSS, JavaScript (or React for better UI)

The system must be production-level, have a clean modern UI, proper folder structure, secure coding practices, and follow MVC architecture if possible.

========================

========================

The Parking Management System allows users to:
- Register and login

- Park/register vehicles
- Track entry and exit times
- View parking history
- Make payments for parking
- Check parking slot availability

Admin can:
- Manage parking slots
- Manage vehicles and parking records
- Manage users
- Monitor occupied/free slots
- View payments
- Generate parking reports and analytics

========================
📊 DATABASE DESIGN (VERY IMPORTANT)
========================

Use the attached ERD as the database reference and implement the schema in MySQL.

Entities and attributes:

1. ParkingLot
   - lotID (PK)
   - lotName
   - capacity
   - location

2. ParkingSlot
   - slotID (PK)
   - slotNumber
   - location
   - status

3. Vehicle
   - vehicleID (PK)
   - licensePlate
   - vehicleType
   - ownerName

4. ParkingRecord
   - recordID (PK)
   - vehicleID (FK)
   - slotID (FK)
   - entryTime
   - exitTime

5. Payment
   - paymentID (PK)
   - recordID (FK)
   - paymentMethod
   - paymentTime
   - amount

Relationships:
- One ParkingLot contains many ParkingSlots (1-M)
- One ParkingSlot can have many ParkingRecords over time (1-M)
- One Vehicle can have many ParkingRecords (1-M)
- One ParkingRecord generates one Payment (1-1)
- Vehicles park in ParkingSlots
- Payments are assigned to parking records

Constraints:
- Enforce foreign keys
- Prevent assigning occupied slots
- Prevent duplicate active parking records for the same vehicle
- Amount must be positive
- License plate should be unique
- Exit time cannot be earlier than entry time
